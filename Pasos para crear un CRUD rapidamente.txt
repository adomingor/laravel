🧠 Opción 2 (recomendada): crear un modelo para la vista

Esto es más limpio y mantenible (sobre todo para proyectos grandes).

📦 1. Crear modelo
php artisan make:model ProductoCategoriaView

📄 2. Modelo app/Models/ProductoCategoriaView.php
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

/**
 * Modelo para la vista vw_productos_categorias
 */
class ProductoCategoriaView extends Model
{
    protected $table = 'vw_productos_categorias';

    protected $primaryKey = 'id';

    public $timestamps = false;

    // Es una vista → no se puede escribir
    public $incrementing = false;

    protected $guarded = [];
}

📄 3. Usarlo en el controlador
use App\Models\ProductoCategoriaView;

public function index()
{
    $productos = ProductoCategoriaView::all();

    return view('productos.index', compact('productos'));
}


🔥 Bonus PRO (paginación + búsqueda)
public function index(Request $request)
{
    $query = ProductoCategoriaView::query();

    if ($request->filled('search')) {
        $query->where('producto', 'ilike', '%' . $request->search . '%');
    }

    $productos = $query->paginate(10);

    return view('productos.index', compact('productos'));
}

-------------- Traducción automática
1. Crear el archivo de traducción (Recomendado)
Si tu aplicación está en español, debes ir a resources/lang/es.json (o crear el archivo si no existe) y agregar la traducción:
json
{
    "Delete": "Eliminar",
    "Show": "Ver",
    "Edit": "Editar",
    "Add new": "Agregar nuevo"
}

2. Usar archivos PHP (Si prefieres carpetas)
Si usas archivos por categorías, verifica en resources/lang/es/messages.php:
php
return [
    'Delete' => 'Eliminar',
];

3. A partir de Laravel 10, y siguiendo en Laravel 12, las traducciones ya no vienen instaladas por defecto para ahorrar espacio. Sin embargo, existe un estándar de la comunidad que es el "complemento" oficial de facto para tener todos los archivos JSON y PHP en español al instante.
La mejor forma de agregarlo es mediante el paquete Laravel Lang:
    1. Instalación rápida (Vía Composer)
Ejecuta este comando en tu terminal para descargar las traducciones de más de 100 idiomas (incluido el español):
bash
composer require laravel-lang/common --dev

    2. Publicar el idioma español
Una vez instalado, ejecuta el comando para que el paquete cree los archivos en tu carpeta lang/:
bash
php artisan lang:add es

    3. Configurar Laravel en Español
Para que Laravel use estos archivos por defecto (y dejes de ver errores de traducción), ve a tu archivo config/app.php y cambia la línea del locale:
php
'locale' => 'es',

Si el error de VS Code persiste:
Si después de hacer esto la extensión de VS Code sigue marcando el error en {{ __('Delete') }}, es porque tú debes agregar esa palabra específica a tu archivo lang/es.json, ya que Laravel Lang trae lo estándar del framework, pero no las palabras propias de tu lógica de negocio:
json
{
    "Delete": "Eliminar",
    "Show": "Ver",
    "Edit": "Editar",
    "Add new": "Agregar nuevo"
}
